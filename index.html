1. Background ¶

This notebook aims to build a predictive model for credit card fraud detection by analyzing a dataset of European cardholders' transactions over a two-day period in September 2013. The dataset contains anonymized transaction details and is highly imbalanced, with a small fraction of transactions labeled as fraudulent. The goal is to explore the data, apply appropriate preprocessing techniques, and evaluate machine learning models to effectively identify fraudulent transactions.

2. Data Description and Cleaning¶

The dataset is publicly available on kaggle and provided by Worldline and the Machine Learning Group of ULB.

This dataset contains 284807 rows and 31 columns. Due to confidentiality issues, the original data have been transformed using PCA. Below is a detailed description of each feature:

  • Time: The number of seconds elapsed between each transcations and the first transaction.
  • V1-V28: Scaled numerical features obtained through PCA.
  • Amount: The transcation amount.
  • Class: The variable indicating whether a transaction is fraudulent:
    • 0: Normal
    • 1: Fraud

4. Data Preprocessing ¶

We perform critical preprocessing steps to improve model performance:

  1. Apply sine and cosine transformation to the hour of day to capture cyclical patterns.
  2. Standardize Amount feature.
  3. Drop the original Time, Hours and Amountcolumns.
In [15]:
df = data.copy()
df['Hours_sin'] = np.sin(2*np.pi*df['Hours']/24)
df['Hours_cos'] = np.cos(2*np.pi*df['Hours']/24)

std_scaler= StandardScaler()
df[['Amount_sca']] = std_scaler.fit_transform(df[['Amount']])

df.drop(['Time','Hours','Amount'], axis=1, inplace=True)

6. Conclusion ¶

We conducted exploratory data analysis on a credit card fraud dataset and built a fraud detection model using XGBoost. The model achieved high performance in terms of precision and reall.

However, there are several areas for improvement in future work: - We identified some potentially irrelecant features but did not remove them which may introduce noise. - The model was not compared with other algorithms. - The model was trained using mostly default hyperparameters, with minimal tuning.

In summary, this project provides a basic framework for fraud detection modeling, which can be improved and expanded upon in future work.
